# 16 编辑器系统

## 1. 设计目标

开发者在游戏内直接修改数据，调整后**即时生效**，不需要修改代码、重启服务或重新部署。

```
传统流程：改代码 → 重启 → 看效果 → 再改代码
编辑器流程：在编辑器里改数值 → 保存 → 立即生效
```

---

## 2. 编辑器构成

三种编辑器，共用一个入口：

```
主菜单 → 编辑器
          ├── 地图编辑器     ← 编辑地图（地形/地貌/河流/起始位置）
          ├── 单位编辑器     ← 编辑单位属性、词条配置
          └── 建筑/改良编辑器 ← 编辑建筑和改良的属性
```

---

## 3. 地图编辑器

### 3.1 用途

创建和编辑游戏地图：绘制地形、放置城市起始点、设置玩家出生位置。

### 3.2 界面布局

```
┌─────────────────────────────────────────────────────┐
│ [工具栏]    地图名称: [维斯特洛_标准]    [保存] [载入] │
├─────────────────────────────────────────────────────┤
│                                                     │
│                                                     │
│          地图画布（六边形网格，可缩放/拖拽）          │
│                                                     │
│  当前笔刷: [平原 ▼]                                 │
│  大小: [1格 ▼]                                      │
│                                                     │
├─────────────────────────────────────────────────────┤
│ [信息栏] 坐标: (12, 8)  地形: 平原  地貌: 无        │
└─────────────────────────────────────────────────────┘
```

### 3.3 功能

| 功能 | 说明 |
|------|------|
| 刷地形 | 选择地形类型，点击/拖拽绘制到地图上 |
| 刷地貌 | 在地形上叠加地貌（受地貌限制矩阵约束） |
| 画河流 | 在六边形边上绘制河流，自动同步相邻格 |
| 清空 | 移除单元格上的地貌 |
| 设置出生点 | 标记某格为玩家/阵营的出生位置 |
| 设置城市起始点 | 标记某格为预设城市位置 |
| 缩放/拖拽 | 浏览整个地图 |
| 保存/载入 | 将地图保存为 JSON 文件，或载入已有地图 |
| 地图大小 | 新建地图时设置 width 和 height |
| 验证 | 检查地图是否合法（如检查玩家出生点数量是否正确） |

### 3.4 数据输出

地图编辑器保存的文件格式：

```json
{
  "map_name": "维斯特洛_标准",
  "width": 80,
  "height": 60,
  "cells": {
    "(12,8)": {
      "terrain": "plain",
      "feature": null,
      "river_mask": 5,
      "bridge_mask": 0
    },
    "(12,9)": {
      "terrain": "hill",
      "feature": "forest",
      "river_mask": 0,
      "bridge_mask": 0
    }
  },
  "spawn_points": {
    "NK": {"q": 5, "r": 10},
    "WL": {"q": 60, "r": 45}
  },
  "preset_cities": [
    {"name": "临冬城", "q": 5, "r": 10, "faction": "NK"}
  ]
}
```

## 4. 单位编辑器

### 4.1 用途

编辑单位的基础属性、攻击类型、词条配置。新增或修改一个单位类型后，游戏中即可招募该单位。

### 4.2 界面设计

```
┌──────────────────────────────────────────────────────┐
│  单位编辑器                     [新增] [保存] [取消]  │
├──────────────────────────────────────────────────────┤
│                                                       │
│  ┌──────────────────┐  ┌──────────────────────────┐  │
│  │  单位列表          │  │  属性编辑                 │  │
│  │                     │  │                          │  │
│  │  ☑ 征召兵          │  │  名称: [征召兵          ] │  │
│  │  ☑ 军士            │  │  ID:   [levy             ] │  │
│  │  ☐ 骑士            │  │  阵营: [全维斯特洛     ▼] │  │
│  │  ☐ 长弓手          │  │                          │  │
│  │  ☐ 无垢者          │  │  max_hp: [60]            │  │
│  │  ☐ 巨人            │  │  attack: [20]            │  │
│  │  ☐ ...             │  │  defense: [10]           │  │
│  │                     │  │  max_move: [3]           │  │
│  │  筛选: [全部 ▼]     │  │  range: [1]              │  │
│  │  搜索: [______]     │  │  attack_type: [MELEE ▼]  │  │
│  │                     │  │                          │  │
│  │                     │  │  ── 词条 ──              │  │
│  │                     │  │  [+ 添加词条]            │  │
│  │                     │  │  ☑ 无甲 (移动消耗无视森林)│  │
│  │                     │  │  ☐ 狂热 (HP<30% 攻击+10) │  │
│  │                     │  │                          │  │
│  └──────────────────┘  └──────────────────────────┘  │
└──────────────────────────────────────────────────────┘
```

### 4.3 功能

| 功能 | 说明 |
|------|------|
| 列表/搜索/筛选 | 按阵营、类型等筛选单位列表 |
| 新增单位 | 填写基础属性，创建新单位类型 |
| 编辑属性 | 修改任何基础属性值 |
| 配置词条 | 从词条库中选择词条，配置参数 |
| 删除单位 | 移除单位类型 |
| 克隆单位 | 以某个单位为基础复制一份，修改为新单位 |
| 预览 | 显示单位最终属性（含词条加成） |

### 4.4 数据流向

```
单位编辑器 → 保存 → PostgreSQL unit_types 表
                      → unit_traits 关联表
                      ↓
              游戏引擎加载配置 → 游戏中可用
```

## 5. 建筑/改良编辑器

### 5.1 用途

编辑建筑和改良的数据：产出值、建造成本、地形限制、前置条件。

### 5.2 界面

```
├──────────────────────────────────────────────────────┤

│  ┌──────────────────┐  ┌──────────────────────────┐  │

│  │                     │  │                          │  │
│  │  ☑ 农场            │  │  类型: 改良               │  │
│  │  ☑ 牧场            │  │  名称: [农场             ] │  │
│  │  ☑ 伐木场          │  │  ID:   [farm             ] │  │
│  │  ☑ 矿井            │  │                          │  │
│  │  ☑ 渔港            │  │  ── 产出 ──              │  │
│  │  ☑ 哨塔            │  │  人力: [+3]              │  │
│  │  ☑ 要塞            │  │                          │  │
│  │  ☐ 市场 (建筑)     │  │  ── 成本 ──              │  │

│  │                     │  │  ☑ 平原                  │  │
│  │                     │  │  ☐ 丘陵                  │  │

```

### 5.3 功能

| 功能 | 说明 |
|------|------|
| 切换建筑/改良 | 在工具栏切换编辑对象类型 |
| 编辑产出 | 修改资源产出值（每种资源独立编辑） |
| 编辑成本 | 修改建造成本（资源类型和数量） |
| 编辑地形限制 | 勾选/取消该设施允许建造的地形 |
| 编辑前置条件 | 设置需要先建造的其他建筑（仅建筑） |
| 预览效果 | 模拟显示一座城市配置该建筑/改良后的产出变化 |

### 5.4 数据流向

```
建筑/改良编辑器 → 保存 → PostgreSQL building_types 表
                              → improvement_types 表
                              ↓
                      游戏引擎加载配置 → 游戏中可用
```

## 6. 编辑器通用规则

| 规则 | 说明 |
|------|------|
| 权限保护 | 编辑器仅在开发者模式下可见，普通玩家不可见 |
| 即时生效 | 修改保存后，新游戏直接使用新数据（已开始的游戏不受影响） |
| 数据导出/导入 | 支持将全部配置导出为 JSON 快照，也可导入恢复 |
| 版本标记 | 每次编辑保存后增加版本号，方便回溯 |
| 冲突提示 | 删除某个单位/建筑时，提示哪些地方引用了它 |

## 7. 设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 三种编辑器分开 | 各管各的数据 | 职责清晰，避免一个编辑器功能过于臃肿 |
| 编辑器在游戏客户端内 | 不单独做 Web 版 | 编辑地图需要实时看到六边形网格，客户端内最方便 |
| 管理后台在 Web | Web 端 CRUD | 配置表管理适合表格操作，不需要图形化地图预览 |
| 单位/建筑编辑器保存即生效 | 写入配置表 | 新游戏直接读最新配置，旧对局不受影响 |
