# 10 建筑概念

## 1. 什么是建筑

建筑是建造在**城市内部**的设施，不占据地图单元格，影响整座城市的维度。

与单元格改良的区分：

| 维度 | 建筑（本文） | 改良（11-单元格改良） |
|------|------------|-------------------|
| 位置 | 城市内部（市中心） | 地图单元格上 |
| 占格 | 不占格 | 占一格 |
| 效果范围 | 整座城市 | 仅所在格（或影响周边） |
| 被摧毁 | 城市被占时可能被毁 | 可被敌方劫掠拆除 |

## 2. 建筑数据结构

```
Building {
    id: BuildingID                   // 唯一标识
    type: BuildingType               // 建筑类型（见第 3 节）
    
    // 效果定义（具体值 [待定]）
    effects: list<BuildingEffect>
    
    // 建造成本（具体值 [待定]）
    cost: dict<ResourceID, int>
    
    // 前置条件
    prerequisites: list<BuildingID>  // 前置建筑
    terrain_required: TerrainType | None  // 地形要求（如港口需沿海）
}
```

## 3. 建筑类型枚举

| 建筑 | ID | 说明 | 分类 |
|------|----|------|------|
| 市场 | market | 提升城市金币产出 | 经济 |
| 军营 | barracks | 提升该城招募单位经验 | 军事 |
| 铁匠铺 | smithy | 提升该城训练单位攻击 | 军事 |
| 马厩 | stable | 允许该城训练骑兵 | 解锁类 |
| 港口 | port | 允许该城建造海军 | 解锁类，需沿海 |
| 城墙 | wall | 提升城市防御和耐久 | 城防 |
| 塔楼 | tower | 提升城市视野范围 | 城防 |
| 仓库 | warehouse | 提升资源存储上限 | 存储 |

> 每类建筑的具体属性值（金币加成量、经验值、建造成本等）[待定]，待单位类型和平衡测试时补充。

---

## 4. 建造规则

- 建筑在**城市内**建造，不占地图格
- 每建造一个建筑消耗资源（资源类型和数量 [待定]）
- 建筑一旦建成永久有效
- 城市被占领时，部分建筑可能被摧毁（概率 [待定]）
- 部分建筑有前置要求（如港口需要城市位于沿海格）

---

## 5. 设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 建筑不占地图格 | 是 | 建筑是城市面板上的属性，与地图坐标无关 |
| 解锁类建筑（港口/马厩） | 是 | 通过建筑解锁兵种，而非通过城市等级 |
