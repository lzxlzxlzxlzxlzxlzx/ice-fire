# 07 战斗逻辑

## 1. 战斗触发

### 1.1 近战攻击条件

近战单位可以对目标发动攻击，当且仅当：
1. 目标与攻击者相邻（hex_distance = 1）
2. 攻击者剩余移动力 ≥ 移动到目标格所需的移动力消耗
3. 攻击者本回合未攻击过
4. 目标为敌方单位

### 1.2 远程攻击条件

远程单位可以对目标发动攻击，当且仅当：
1. 目标在射程内（hex_distance ≤ range）
2. 攻击者本回合未攻击过
3. 目标为敌方单位
4. 攻击者与目标之间无视线遮挡（待 02-单元格类型 定义遮挡规则）

> 视线检查使用 hex_line 算法（见 01-地图基础概念 3.5），路径上存在遮挡地形的格子则视线被阻断。

## 2. 伤害计算

### 2.1 核心公式

```
伤害 = max(1, 攻击方攻击力 - 防御方防御力)
```

- 攻击力 = 单位基础攻击力 + 词条加成 + 地形加成 + 状态效果加成
- 防御力 = 单位基础防御力 + 词条加成 + 地形加成 + 状态效果加成
- 伤害下限为 1，不会出现 0 伤害或负伤害

### 2.2 远程被近战攻击时的反击削弱

当远程单位遭受近战攻击时，其反击使用的攻击力：

```
反击攻击力 = 远程单位攻击力 - 10
```

此削弱仅影响反击伤害计算，不影响远程单位主动攻击时的攻击力。

若反击攻击力 ≤ 防御方防御力，反击伤害仍保底 1 点。

## 3. 近战攻击流程

```
近战攻击(attacker, defender):
    // 保存攻击者原始位置
    origin = (attacker.q, attacker.r)
    
    // === 第一步：攻击者对防守方造成伤害 ===
    damage_to_defender = max(1, attacker.attack - defender.defense)
    defender.hp -= damage_to_defender
    
    // === 第二步：防守方反击 ===
    counter_attack = defender.attack
    if defender.attack_type == RANGED:
        counter_attack = defender.attack - 10
    damage_to_attacker = max(1, counter_attack - attacker.defense)
    attacker.hp -= damage_to_attacker
    
    // === 第三步：结算战斗结果 ===
    结算(attacker, defender, origin)
    
    // === 第四步：攻击者状态更新 ===
    attacker.has_attacked = true
    attacker.move_points = 0
```

---

## 4. 远程攻击流程

```
远程攻击(attacker, defender):
    // 远程攻击不移动
    
    // === 第一步：攻击者对目标造成伤害 ===
    damage_to_defender = max(1, attacker.attack - defender.defense)
    defender.hp -= damage_to_defender
    
    // === 第二步：无反击 ===
    // 远程攻击不触发任何反击
    
    // === 第三步：结算战斗结果 ===
    结算(attacker, defender, null)
    
    // === 第四步：攻击者状态更新 ===
    attacker.has_attacked = true
    attacker.move_points = 0
```

## 5. 战斗结果结算

```
结算(attacker, defender, attacker_origin):
    
    attacker_alive = attacker.hp > 0
    defender_alive = defender.hp > 0
    
    if defender_alive and attacker_alive:
        // 双方存活：攻击者回到原位
        // （近战攻击时，攻击者不移动；远程攻击无此问题）
        // 无额外处理，攻击者留在原位
        
    elif not defender_alive and attacker_alive:
        // 防守方死亡，攻击方存活：攻击者移动到防守方位置
        if attacker.attack_type == MELEE:
            移动单位(attacker, defender.q, defender.r)
        移除单位(defender)
        
    elif defender_alive and not attacker_alive:
        // 攻击方死亡，防守方存活
        移除单位(attacker)
        
    elif not defender_alive and not attacker_alive:
        // 双方均死亡（生命值 ≤ 0）
        if attacker.hp < defender.hp:
            // 攻击者负值更大，攻击者死亡
            // 防守者负值较小，存活，HP 恢复为 1
            defender.hp = 1
            移除单位(attacker)
            if attacker.attack_type == MELEE:
                // 攻击者已死，不移动
                pass
        elif attacker.hp > defender.hp:
            // 防守者负值更大，防守者死亡
            // 攻击者负值较小，存活，HP 恢复为 1
            attacker.hp = 1
            移除单位(defender)
            if attacker.attack_type == MELEE:
                移动单位(attacker, defender.q, defender.r)
        else:
            // 负值相等（平局）：防守方存活
            defender.hp = 1
            移除单位(attacker)
```

### 5.1 结算规则总结

| 情况 | 攻击者 | 防守者 | 攻击者位置 |
|------|--------|--------|-----------|
| 双方存活 | 存活 | 存活 | 不移动（留在原位） |
| 防守方死亡 | 存活 | 死亡 | 近战：移至防守方位置；远程：不动 |
| 攻击方死亡 | 死亡 | 存活 | — |
| 双方死亡，攻击者负值更大 | 死亡 | 存活(hp=1) | — |
| 双方死亡，防守者负值更大 | 存活(hp=1) | 死亡 | 近战：移至防守方位置；远程：不动 |
| 双方死亡，负值相等 | 死亡 | 存活(hp=1) | — |

## 6. 反击规则汇总

| 攻击类型 → 防守类型 | 防守方是否反击 | 反击攻击力 |
|---------------------|---------------|-----------|
| 近战 → 近战 | 是 | 防守方攻击力（正常值） |
| 近战 → 远程 | 是 | 防守方攻击力 - 10 |
| 远程 → 近战 | 否 | — |
| 远程 → 远程 | 否 | — |

## 7. 攻击与移动的关系

### 7.1 移动后攻击

单位可以先移动再攻击：
1. 消耗移动力移动到目标位置
2. 从新位置判断是否满足攻击条件
3. 发动攻击
4. 攻击后剩余移动力归零

### 7.2 攻击后移动

攻击后不可再移动：
- 攻击 → move_points = 0 → 本回合不可再移动

### 7.3 多次攻击

每回合最多攻击一次：
- has_attacked = true 后不可再次攻击
- 下一回合开始时重置 has_attacked = false

## 8. 战斗逻辑示例

### 示例1：步兵(近战) vs 弓手(远程)

```
步兵：hp=100, attack=30, defense=20
弓手：hp=70, attack=35, defense=10

步兵攻击弓手：
  步兵对弓手伤害 = max(1, 30 - 10) = 20
  弓手 hp = 70 - 20 = 50
  
  弓手反击（攻击力-10）：
  反击攻击力 = 35 - 10 = 25
  弓手对步兵伤害 = max(1, 25 - 20) = 5
  步兵 hp = 100 - 5 = 95
  
  双方存活，步兵留在原位。
  步兵 move_points = 0。
```

### 示例2：骑兵(近战) vs 步兵(近战)，骑兵击杀步兵

```
骑兵：hp=90, attack=40, defense=15
步兵：hp=30, defense=20  // 已受伤的步兵

骑兵攻击步兵：
  骑兵对步兵伤害 = max(1, 40 - 20) = 20
  步兵 hp = 30 - 20 = 10
  
  步兵反击：
  步兵对骑兵伤害 = max(1, 30 - 15) = 15
  骑兵 hp = 90 - 15 = 75
  
  双方存活，骑兵留在原位。
  （步兵未死亡，骑兵不移动到步兵位置）
```

### 示例3：双方同归于尽（负值比较）

```
单位A：hp=5, attack=50, defense=5
单位B：hp=5, attack=50, defense=5

A攻击B：
  A对B伤害 = max(1, 50 - 5) = 45
  B hp = 5 - 45 = -40
  
  B反击：
  B对A伤害 = max(1, 50 - 5) = 45
  A hp = 5 - 45 = -40
  
  双方死亡，负值相等(-40 == -40)，防守方存活：
  B hp = 1，A 被移除。
```

### 示例4：远程攻击

```
弓手：hp=70, attack=35, defense=10, range=3
步兵：hp=100, attack=30, defense=20

弓手攻击步兵（距离2）：
  弓手对步兵伤害 = max(1, 35 - 20) = 15
  步兵 hp = 100 - 15 = 85
  
  无反击。
  弓手不移动。
  弓手 move_points = 0。
```

## 10. 设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 伤害公式 | 攻击力 - 防御力 | 简洁，防御有明确意义 |
| 伤害保底 | 1 | 防止高防完全免伤 |
| 远程被近战反击削弱 | 攻击力-10 | 近战克制远程，但远程不是完全无力 |
| 远程被远程打 | 无反击 | 远程对远程是单向优势（先手方有利） |
| 攻击后移动力归零 | 是 | 控制节奏，防止单位攻击后继续跑 |
| 同归于尽时负值比较 | 负值小者存活(hp=1) | 避免绝对同归于尽的沮丧感 |
| 平局时防守方存活 | 是 | 防守方有位置优势 |
