# 04 单位属性

## 1. 基础属性

所有单位共享以下基础属性字段：

| 属性 | 字段名 | 类型 | 说明 |
|------|--------|------|------|
| 生命值 | hp | int | 当前生命值，≤0 时单位死亡 |
| 最大生命值 | max_hp | int | 生命上限 |
| 攻击力 | attack | int | 决定造成的伤害 |
| 防御力 | defense | int | 决定减免的伤害 |
| 移动力 | move_points | int | 当前剩余移动力，每回合恢复至上限 |
| 移动力上限 | max_move_points | int | 每回合恢复的移动力上限 |

### 1.1 属性计算规则

最终属性值 = 基础值 + 词条加成 + 地形加成 + 状态效果加成

计算顺序：
1. 取基础值（单位类型的默认值）
2. 叠加永久性词条加成（装备、阵营特性等）
3. 叠加地形加成（如森林+防御）
4. 叠加临时状态效果（如增益/减益buff）

> 各加成为加法叠加，最终结果无上限。伤害计算时再应用保底规则。

## 2. 攻击类型

| 攻击类型 | 枚举值 | 说明 |
|----------|--------|------|
| 近战 | MELEE | 攻击范围1，攻击后移至目标格（若击杀），触发反击 |
| 远程 | RANGED | 攻击范围 >1，不移动，被近战攻击时反击攻击力-10 |

每个单位有固定攻击类型，不随战斗状态变化。

---

## 3. 射程

| 属性 | 字段名 | 类型 | 说明 |
|------|--------|------|------|
| 射程 | range | int | 攻击的最大六边形距离 |

- 近战单位：range = 1（即相邻格）
- 远程单位：range ≥ 2，具体值由单位类型定义

射程判定使用 `hex_distance` 计算（见 01-地图基础概念 3.2），需满足 `distance(attacker, target) ≤ range`。

> 远程攻击是否受视线遮挡影响，待 07-战斗逻辑 中进一步定义。

---

## 4. 单位状态

### 4.1 移动状态

| 状态 | 条件 | 效果 |
|------|------|------|
| 可移动 | move_points > 0 且本回合未攻击 | 可以消耗移动力移动 |
| 可攻击 | 本回合未攻击 | 可以发动攻击（需满足攻击条件） |
| 已行动 | 本回合已攻击 | 失去全部剩余移动力，不可再移动或攻击 |

### 4.2 生命状态

| 状态 | 条件 | 效果 |
|------|------|------|
| 存活 | hp > 0 | 正常参与游戏 |
| 死亡 | hp ≤ 0 | 单位从地图移除 |

## 5. 单位数据结构

```
Unit {
    id: UnitID                           // 唯一标识
    type: UnitTypeID                     // 单位类型（见 03-单位类型）
    faction: FactionID                   // 所属阵营
    
    // 基础属性
    max_hp: int                          // 最大生命值
    hp: int                              // 当前生命值
    attack: int                          // 攻击力
    defense: int                         // 防御力
    max_move_points: int                 // 移动力上限
    move_points: int                     // 当前剩余移动力
    range: int                           // 射程
    attack_type: AttackType              // MELEE / RANGED
    
    // 位置
    q: int                               // 当前所在格 q 坐标
    r: int                               // 当前所在格 r 坐标
    
    // 回合状态
    has_attacked: bool                   // 本回合是否已攻击
    
    // 词条与状态
    traits: list<TraitID>                // 单位拥有的词条列表
    status_effects: list<StatusEffect>   // 当前生效的状态效果列表
}
```

### 5.1 StatusEffect 结构

```
StatusEffect {
    effect_id: string                    // 效果标识
    remaining_turns: int                 // 剩余持续回合数
    attack_modifier: int                 // 攻击力修正（可为负）
    defense_modifier: int                // 防御力修正（可为负）
    move_modifier: int                   // 移动力修正（可为负）
}
```

## 7. 设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 属性加法叠加 | 加法 | 简单直观，便于计算和调试 |
| 伤害保底 | 1点 | 避免高防单位完全免伤 |
| 攻击后移动力归零 | 是 | 防止单位攻击后继续移动，控制节奏 |
| 远程反击惩罚 | 攻击力-10 | 近战克制远程，但不是完全无力反击 |
