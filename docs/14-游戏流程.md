# 14 游戏流程

## 1. 一局游戏的完整生命周期

```
启动游戏 → 主菜单 → 游戏设置 → 游戏中 → 游戏结束 → 返回主菜单
                                              ↓
                                         保存回放 / 查看统计
```

---

## 2. 阶段一：游戏设置

### 2.1 游戏模式

| 模式 | 说明 |
|------|------|
| 单人模式 | 玩家 vs AI（可设置 AI 数量和难度） |
| 多人模式 | 玩家 vs 玩家（热座/网络） |
| AI 沙箱 | AI vs AI（仅开发者/管理员使用，用于平衡测试） |

### 2.2 设置项

- **地图选择**：选择预设地图或随机生成
- **阵营选择**：玩家选择阵营（见 12-阵营概念）
- **AI 数量与难度**：AI 对手的数量和策略等级
- **高级设置**：地图大小、资源倍率、是否启用特殊规则

### 2.3 开局条件

```
所有玩家准备就绪 → 地图初始化 → 分配初始单位 → 游戏开始
```

- 地图初始化：加载地图模板，生成初始地形
- 初始单位：每个玩家获得 [待定] 个初始单位（通常是 1 个开拓者 + 1 个初始部队）
- 初始城市：可选（从开拓者建立第一个城市开始，或开局直接拥有首都）

---

## 3. 阶段二：游戏中（回合制循环）

### 3.1 回合结构

每回合按以下顺序执行：

```
┌─────────────────────────────────────────────────┐
│              回合开始阶段                         │
│  ├─ 城市产出结算（基础 + 建筑 + 改良）            │
│  ├─ 单位维护费扣除                               │
│  ├─ 城市 HP 自动恢复                             │
│  ├─ 城墙修复检查（3 回合未受损 + 消耗石材）        │
│  ├─ 单位移动力重置                               │
│  ├─ 单位 has_attacked 重置                       │
│  ├─ 城墙 has_attacked 重置                       │
│  ├─ buff/debuff 计时更新                         │
│  └─ 城市人口增长                                 │
├─────────────────────────────────────────────────┤
│              玩家行动阶段                         │
│  ├─ 玩家 1 行动                                  │
│  │    ├─ 移动单位                                │
│  │    ├─ 攻击                                    │
│  │    ├─ 建造改良                                │
│  │    ├─ 建造建筑                                │
│  │    ├─ 招募单位                                │
│  │    ├─ 城墙射击                                │
│  │    └─ 结束回合                                │
│  ├─ 玩家 2 行动                                  │
│  │    └─ ...                                      │
│  ├─ 玩家 N 行动                                  │
│  │    └─ ...                                      │
├─────────────────────────────────────────────────┤
│              回合结束阶段                         │
│  ├─ 检查胜利条件                                 │
│  ├─ 自动存档（可选）                              │
│  └─ 回合数 +1 → 进入下一回合                      │
└─────────────────────────────────────────────────┘
```

### 3.2 玩家行动阶段的具体操作

玩家的回合内可以执行以下操作，**顺序不限**，每个单位每回合只能攻击一次：

| 操作 | 说明 | 限制 |
|------|------|------|
| 移动 | 选择单位，点击目标格移动 | 消耗移动力，移动后不可再移动 |
| 攻击 | 选择单位，攻击范围内的敌方目标 | 每单位每回合一次，攻击后移动力归零 |
| 建造改良 | 选择己方城市边界内的空格，建造改良 | 消耗城市资源 |
| 建造建筑 | 在城市面板中选择建筑建造 | 消耗城市资源，需满足前置条件 |
| 招募单位 | 在城市面板中选择单位招募 | 消耗城市资源，单位出现在城市格 |
| 城墙射击 | 有城墙的城市执行一次远程射击 | 每城每回合一次 |

### 3.3 回合切换条件

玩家点击"结束回合"按钮 → 轮到下一玩家 → 所有玩家完成后 → 进入下一回合。

**超时机制**（多人模式）：[待定] 秒未操作自动结束回合。

---

## 4. 阶段三：游戏结束

### 4.1 胜利条件

| 条件 | 判定 | 说明 |
|------|------|------|
| 征服胜利 | 某阵营消灭所有敌对阵营 | 标准胜利方式 |
| 投降 | 某阵营主动认输 | 多人/对 AI 可用 |

> 其他胜利方式（如占领特定目标点、分数胜利等）[待定]，后续可按需添加。

### 4.2 结算

```
胜利条件达成 → 显示胜利/战败画面
              ├── 查看统计数据（击杀数、城市数、回合数等）
              ├── 保存回放（可复盘）
              └── 返回主菜单
```

## 6. 设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 先结算再行动 | 回合开始阶段先算产出和恢复 | 玩家行动时看到的是最新资源数 |
| 操作顺序不限 | 可先移动后攻击，也可先攻击后移动（攻击后移动力归零） | 给玩家战术自由度 |
| AI 和人类共用同一行动框架 | 是 | AI 的每一次"决策"映射为一个玩家操作 |
