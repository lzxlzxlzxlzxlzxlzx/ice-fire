# 02 单元格类型

## 1. 单元格结构

每个单元格由三层数据叠加组成：

```
单元格 = 基础地形 + 地貌（可选） + 边修饰（0~6 条边）

  ┌──────────────┐
  │  基础地形    │  ← 不可改变（地图生成时确定）
  ├──────────────┤
  │  地貌        │  ← 可选叠加，可被砍伐/移除
  ├──────────────┤
  │  边修饰      │  ← 格子 6 条边上的修饰（河流、桥梁）
  └──────────────┘
```

## 3. 地貌（Feature）

### 3.1 地貌枚举

共 5 种地貌，叠加在基础地形之上：

| 编号 | 地貌 | ID | 可叠加的地形 | 可否移除 | 移除产物 |
|------|------|----|-------------|---------|---------|
| F01 | 森林 | forest | 平原、丘陵 | 是（砍伐） | 木材 |
| F02 | 雨林 | rainforest | 平原、丘陵 | 是（砍伐，耗时 2 倍） | 木材 |
| F03 | 沼泽 | swamp | 平原、丘陵 | 否 | — |
| F04 | 绿洲 | oasis | **仅沙漠** | 否 | — |
| F05 | 废墟 | ruin | 平原、丘陵、沙漠、荒地 | 否（可探索一次性） | 随机 |

### 3.2 地貌对属性的影响

地貌**修正**基础地形的战斗和移动属性（加法叠加）：

| 地貌 | 移动消耗修正 | 防御加成修正 | 额外阻挡视线 |
|------|------------|-------------|-------------|
| 森林 | +1 | +2 | **是** |
| 雨林 | +2 | +1 | **是** |
| 沼泽 | +2 | -1 | 否 |
| 绿洲 | 0 | 0 | 否 |
| 废墟 | 0 | 0 | 否 |

**最终单元格属性计算公式：**

```
最终移动消耗 = 地形移动消耗 + 地貌移动修正（如有）
最终防御加成 = 地形防御加成 + 地貌防御修正（如有）
```

> 资源产出已移至城市层面统一计算（见 09-城市概念），单元格不再独立产出资源。
→ 最终：移动 2，人力产出 2
```

## 4. 边修饰（Edge Modifier）

### 4.1 设计原理

边修饰是附加在**六边形边上**的数据，而非格子上。每个格子有 6 条边，每条边可以有 0~1 个修饰。

```
        边 2         边 1
            ╱───  ───╲
           ╱           ╲
    边 3  (    格子     ) 边 0
           ╲           ╱
            ╲───  ───╱
        边 4         边 5
```

**方向编号与对向关系（与 01-地图基础概念 一致）：**

| 方向编号 | 名称 | 向量 | 对向 |
|---------|------|------|------|
| 0 | E | (+1, 0) | 3 |
| 1 | NE | (+1, -1) | 4 |
| 2 | NW | (0, -1) | 5 |
| 3 | W | (-1, 0) | 0 |
| 4 | SW | (-1, +1) | 1 |
| 5 | SE | (0, +1) | 2 |

### 4.2 边修饰枚举

| 编号 | 边修饰 | ID | 说明 |
|------|--------|----|------|
| E01 | 河流 | river | 两个格子之间的水域边界 |
| E02 | 桥梁 | bridge | 跨越河流的通道，抵消河流惩罚 |

### 4.3 边修饰效果

| 边修饰 | 移动影响 | 战斗影响 | 备注 |
|--------|---------|---------|------|
| 河流 | 跨河额外 +2 移动力 | 跨河进攻攻击力 -5 | 不可在河流格停留 |
| 桥梁 | 无视河流惩罚 | 无影响 | 必须在有河流的边上建造 |

### 4.4 河流一致性约束

河流是共享数据——格子 A 的东边有河，则格子 B 的西边必须有同一条河：

```
格子 A                         格子 B
┌────────────────┐        ┌────────────────┐
│                │  河流   │                │
│   edge[0] = 1  │════════╡  edge[3] = 1  │
│                │ (E方向) │  (W方向)       │
└────────────────┘        └────────────────┘
```

**一致性规则：**

```
has_river(cell_a, direction) == has_river(cell_b, (direction + 3) % 6)
其中 cell_b = neighbor(cell_a, direction)
```

**地图生成后的同步校验：**

```
sync_river_edges(map):
    for each (q, r) in map.cells:
        for direction in [0..5]:
            neighbor = get_neighbor(q, r, direction)
            if neighbor not in map.cells: continue
            if has_river(map.cells[(q, r)], direction):
                set_river(map.cells[neighbor], (direction + 3) % 6)
            if has_river(map.cells[neighbor], (direction + 3) % 6):
                set_river(map.cells[(q, r)], direction)
```

### 4.5 跨河判断逻辑

**移动时：**

```
can_move_between(from_cell, to_cell) → bool:
    direction = get_direction(from_cell, to_cell)
    if has_river(from_cell, direction):
        remaining_move = current_unit.move_points - 2  # 额外消耗
        if remaining_move < 0: return false
        # 还要跨过桥则抵消惩罚
        if has_bridge(from_cell, direction): return true
    return true
```

**攻击时：**

```
get_attack_modifier(attacker_cell, defender_cell) → int:
    direction = get_direction(attacker_cell, defender_cell)
    if has_river(attacker_cell, direction):
        if not has_bridge(attacker_cell, direction):
            return -5  # 跨河进攻惩罚
    return 0
```

## 5. 数据结构

### 5.1 单元格完整数据

```python
class Cell:
    # 坐标
    q: int
    r: int
    
    # 基础属性
    active: bool                    # 是否存在于地图中
    terrain: TerrainType            # 基础地形（plain, grassland, hill...）
    feature: FeatureType | None     # 地貌（forest, swamp, ruin...）
    
    # 边修饰（6 条边，每位代表一条边）
    # bit 0 = E, bit 1 = NE, bit 2 = NW, bit 3 = W, bit 4 = SW, bit 5 = SE
    river_mask: int                 # 6-bit 掩码，标记哪些边有河流
    bridge_mask: int                # 6-bit 掩码，标记哪些边有桥梁
    
    # 所有权
    owner: FactionID | None         # 所属阵营（null = 中立）
    city_id: CityID | None          # 所在城市（null = 非城市格）
    
    # 单位
    unit_ids: list[UnitID]          # 该格上的单位列表
    
    # 改良设施（见 11-单元格改良）
    improvement: ImprovementID | None
    
    # 战争迷雾
    visible_by: set[FactionID]      # 当前可见此格的阵营
    explored_by: set[FactionID]     # 曾经探索过的阵营
```

### 5.3 边修饰辅助函数

```python
# 方向常量
DIR_RIVER_MASK = {
    0: 0b000001,  # E    → bit 0
    1: 0b000010,  # NE   → bit 1
    2: 0b000100,  # NW   → bit 2
    3: 0b001000,  # W    → bit 3
    4: 0b010000,  # SW   → bit 4
    5: 0b100000,  # SE   → bit 5
}

def has_river(cell: Cell, direction: int) -> bool:
    return bool(cell.river_mask & DIR_RIVER_MASK[direction])

def set_river(cell: Cell, direction: int):
    cell.river_mask |= DIR_RIVER_MASK[direction]

def has_bridge(cell: Cell, direction: int) -> bool:
    return bool(cell.bridge_mask & DIR_RIVER_MASK[direction])

def set_bridge(cell: Cell, direction: int):
    cell.bridge_mask |= DIR_RIVER_MASK[direction]
```

## 6. 单元格属性查询

### 6.1 移动消耗查询

```python
def get_move_cost(cell: Cell) -> int:
    """进入该格的基础移动消耗"""
    cost = TERRAIN_MOVE_COST[cell.terrain]
    if cell.feature:
        cost += FEATURE_MOVE_MOD[cell.feature]
    return cost

def get_cross_cost(from_cell: Cell, to_cell: Cell) -> int:
    """从 A 移动到 B 的总消耗（含跨河惩罚）"""
    cost = get_move_cost(to_cell)
    
    direction = get_direction(from_cell, to_cell)
    if has_river(from_cell, direction) and not has_bridge(from_cell, direction):
        cost += 2  # 跨河额外消耗
    
    return cost
```

### 6.2 防御加成查询

```python
def get_defense_bonus(cell: Cell) -> int:
    bonus = TERRAIN_DEFENSE_BONUS[cell.terrain]
    if cell.feature:
        bonus += FEATURE_DEFENSE_MOD[cell.feature]

## 8. 设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 地形 vs 地貌分层 | 两层独立叠加 | 避免组合爆炸（12 地形 × 5 地貌 = 60 种情况，用修正值处理） |
| 河流作为边修饰 | 6-bit 掩码 | 不占格子，跨河逻辑清晰，文明系列同方案 |
| 河流一致性 | 生成后全局同步 | 一次校验保证永远一致，运行期无需再检查 |
| 地貌限制矩阵 | 白名单制 | 防止地图生成时出现不合理组合（如沙漠+森林） |
| 属性计算 | 地形基础 + 地貌修正 | 加法叠加，可单独调整任意维度 |
| 山脉阻挡视线 | 是 | 战术上制造视野盲区，增加策略性 |
| 森林/雨林阻挡视线 | 是 | 伏击战术的基础机制 |
| 沼泽减防御不减移动消耗 | 特殊规则 | 移动惩罚已体现通行困难，防御惩罚体现无法构筑工事 |
| 单元格无资源产出 | 是 | 产出移至城市层面统一计算（见 09-城市概念） |
