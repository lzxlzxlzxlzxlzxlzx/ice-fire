# 01 地图基础概念

## 1. 坐标系

### 1.1 坐标类型：Axial（轴坐标）

采用 Axial 坐标系 (q, r)，定义如下：

- **q 轴**：向右上方延伸
- **r 轴**：向右下方延伸
- 隐含的 s 轴：s = -q - r（不存储，按需计算）

约束：q + r + s = 0

### 1.2 六边形朝向：尖顶（Pointy-Top）

尖顶六边形的几何特征：
- 顶点在正上方和正下方
- 宽度 width = √3 × size
- 高度 height = 2 × size
- size = 外接圆半径（中心到顶点的距离）

尖顶六边形的4个关键尺寸：

| 名称 | 公式 | 说明 |
|------|------|------|
| size | — | 外接圆半径（设计基准值） |
| width | √3 × size | 六边形宽度 |
| height | 2 × size | 六边形高度 |
| inner_radius | √3/2 × size | 内切圆半径（中心到边中点） |

## 2. 方向定义

### 2.1 六个方向

尖顶六边形的方向编号与向量：

| 方向编号 | 名称 | 向量 (dq, dr) | 大致方位 |
|----------|------|----------------|----------|
| 0 | E (东) | (+1, 0) | 右 |
| 1 | NE (东北) | (+1, -1) | 右上 |
| 2 | NW (西北) | (0, -1) | 左上 |
| 3 | W (西) | (-1, 0) | 左 |
| 4 | SW (西南) | (-1, +1) | 左下 |
| 5 | SE (东南) | (0, +1) | 右下 |

```
        NW(0,-1)  NE(+1,-1)
            \      /
             \    /
    W(-1,0) ——  —— E(+1,0)
             /    \
            /      \
        SW(-1,+1)  SE(0,+1)
```

### 2.2 方向运算

**对向（相反方向）**：direction ⊕ 3（即 (i + 3) % 6）

| 方向 | 对向 |
|------|------|
| E (0) | W (3) |
| NE (1) | SW (4) |
| NW (2) | SE (5) |

**顺时针旋转60°**：direction + 1（mod 6）

**逆时针旋转60°**：direction - 1（mod 6）

## 3. 核心算法

### 3.1 邻居计算

```
neighbor(q, r, direction) = (q + DIRECTIONS[direction].dq, r + DIRECTIONS[direction].dr)
```

获取全部6个邻居：

```
all_neighbors(q, r) = [(q+dq, r+dr) for (dq, dr) in DIRECTIONS]
```

### 3.2 距离计算

两格之间的六边形距离：

```
distance(q1, r1, q2, r2) = (|q1 - q2| + |r1 - r2| + |(q1 + r1) - (q2 + r2)|) / 2
```

等价形式（利用 s = -q - r）：

```
distance = max(|q1-q2|, |r1-r2|, |s1-s2|)
```

### 3.3 范围查询

距离中心 (cq, cr) 不超过 N 的所有格子：

```
hex_range(cq, cr, N):
    results = []
    for dq in [-N, ..., +N]:
        for dr in [max(-N, -dq-N), ..., min(+N, -dq+N)]:
            results.append((cq + dq, cr + dr))
    return results
```

格子总数 = 3N² + 3N + 1

| 半径 N | 格子数 |
|--------|--------|
| 1 | 7 |
| 2 | 19 |
| 3 | 37 |
| 4 | 61 |
| 5 | 91 |

### 3.4 环查询

距离中心恰好 N 的环上格子（共 6N 个，N>0 时）：

```
hex_ring(cq, cr, N):
    if N == 0: return [(cq, cr)]
    results = []
    (q, r) = (cq, cr) + DIRECTIONS[4] × N  // 从西南角出发
    for i in [0..5]:
        for j in [0..N-1]:
            results.append((q, r))
            (q, r) = (q, r) + DIRECTIONS[i]
    return results
```

### 3.5 直线算法

从 (q1, r1) 到 (q2, r2) 的直线格子序列（用于视线检查、射击线）：

```
hex_line(q1, r1, q2, r2):
    N = distance(q1, r1, q2, r2)
    if N == 0: return [(q1, r1)]
    results = []
    // 在 cube 坐标下线性插值，然后取整
    for i in [0..N]:
        t = i / N
        // cube 坐标插值
        x = lerp(q1, q2, t)
        y = lerp(r1, r2, t)
        z = lerp(-q1-r1, -q2-r2, t)
        // cube 取整（确保 x+y+z=0）
        (rx, ry, rz) = cube_round(x, y, z)
        results.append((rx, rz))  // axial = (x, z)
    return results

cube_round(x, y, z):
    (rx, ry, rz) = (round(x), round(y), round(z))
    x_diff = |rx - x|
    y_diff = |ry - y|
    z_diff = |rz - z|
    if x_diff > y_diff and x_diff > z_diff:
        rx = -ry - rz
    elif y_diff > z_diff:
        ry = -rx - rz
    else:
        rz = -rx - ry
    return (rx, ry, rz)
```

## 4. 坐标转换

### 4.1 Axial ↔ Offset（奇数行偏移，尖顶六边形）

渲染时需要将 Axial 转为屏幕行列。采用"奇数 r 行右偏"方案：

**Axial → Offset：**

```
col = q + (r - (r & 1)) / 2    // 注意：整数除法，r&1 取 r 的奇偶
row = r
```

**Offset → Axial：**

```
q = col - (row - (row & 1)) / 2
r = row
```

> & 为按位与，r & 1 等价于 r % 2（取奇偶）

### 4.2 Axial → 像素坐标（尖顶六边形）

已知 size（外接圆半径），计算格子中心的像素坐标：

```
pixel_x = size × (√3 × q + √3/2 × r)
pixel_y = size × (3/2 × r)
```

### 4.3 像素坐标 → Axial

```
q = (√3/3 × pixel_x - 1/3 × pixel_y) / size
r = (2/3 × pixel_y) / size
// 然后执行 cube_round 取整到最近的格子
```

## 5. 地图边界

### 5.1 矩形边界定义

地图在 Axial 坐标下定义一个矩形范围：

```
MapBoundary {
    q_min: int    // q 最小值
    q_max: int    // q 最大值
    r_min: int    // r 最小值
    r_max: int    // r 最大值
}
```

格子 (q, r) 是否在地图内：

```
is_in_map(q, r) = (q_min ≤ q ≤ q_max) and (r_min ≤ r ≤ r_max)
```

> 注意：矩形边界在六边形地图上不是完美矩形，角落可能被截断。这属于正常现象，实际地图形状由单元格是否被激活决定（见 5.3）。

### 5.2 地图尺寸参数

| 参数 | 类型 | 说明 |
|------|------|------|
| width | int | 地图宽度（q 方向格数） |
| height | int | 地图高度（r 方向格数） |
| q_min | int | = 0 |
| q_max | int | = width - 1 |
| r_min | int | = 0 |
| r_max | int | = height - 1 |

预设地图尺寸（后续可调整）：

| 名称 | width | height | 总格数（约） | 用途 |
|------|-------|--------|-------------|------|
| 小型 | 40 | 30 | ~1200 | 快速对局 / 测试 |
| 中型 | 80 | 60 | ~4800 | 标准对局 |
| 大型 | 120 | 90 | ~10800 | 大地图长局 |

### 5.3 单元格激活标记

并非矩形范围内的每个格子都是可用的。地图数据中每个格子有 active 标记：

- active = true：该格子存在于地图中，有地形数据
- active = false：该格子不存在（海洋深处、地图外等），不可进入

这允许在矩形边界内创建不规则的海岸线、岛屿等。

## 6. 地图数据结构

### 6.1 地图主结构

```
GameMap {
    width: int                          // 地图宽度
    height: int                         // 地图高度
    cells: dict<(q, r), Cell>           // 所有激活的格子，key 为 (q, r)
}
```

### 6.2 单元格结构

```
Cell {
    q: int                              // Axial q 坐标
    r: int                              // Axial r 坐标
    active: bool                        // 是否存在于地图中
    terrain_type: TerrainType           // 地形类型（见 02 单元格类型）
    terrain_modifiers: list<TerrainModifier>  // 地形修饰（河流、桥梁等）
    owner: FactionID | null             // 所属阵营（null = 中立）
    city_id: CityID | null              // 所在城市（null = 非城市格）
    unit_ids: list<UnitID>              // 该格上的单位列表
    visible_by: set<FactionID>          // 当前可见此格的阵营（战争迷雾）
    explored_by: set<FactionID>         // 曾经探索过的阵营（已探索但可能不可见）
}
```

### 6.3 存储说明

- cells 使用字典（哈希表）存储，key 为 (q, r) 元组
- 优点：不需要为 inactive 格子分配内存；查询 O(1)
- 替代方案：如果需要极致性能，可用一维数组 + index(q, r) = q + r × width 做偏移映射

## 7. 算法补充

### 7.1 寻路算法

移动范围计算：BFS / Dijkstra，考虑地形移动力消耗。

```
reachable_cells(start_q, start_r, move_points):
    // Dijkstra 变体，以移动力为代价
    frontier = PriorityQueue()
    frontier.put((start_q, start_r), 0)
    cost_so_far = {(start_q, start_r): 0}
    results = {(start_q, start_r)}

    while not frontier.empty():
        (q, r) = frontier.get()
        for neighbor in all_neighbors(q, r):
            if not is_in_map(neighbor): continue
            if not cells[neighbor].active: continue
            move_cost = terrain_move_cost(cells[neighbor].terrain_type)
            new_cost = cost_so_far[(q, r)] + move_cost
            if new_cost <= move_points and (neighbor not in cost_so_far or new_cost < cost_so_far[neighbor]):
                cost_so_far[neighbor] = new_cost
                frontier.put(neighbor, new_cost)
                results.add(neighbor)

    return results
```

### 7.2 视线检查

判断从 A 格能否看到 B 格（用于射击、侦察）：

```
has_line_of_sight(q1, r1, q2, r2):
    line = hex_line(q1, r1, q2, r2)
    for each cell in line[1:-1]:  // 跳过起点和终点
        if cells[cell].terrain_type.blocks_sight:
            return false
    return true
```

### 7.3 区域填充

从指定格子出发，泛洪填充满足条件的连续区域（用于领土计算等）：

```
flood_fill(start_q, start_r, condition_fn):
    visited = set()
    queue = [(start_q, start_r)]
    while queue:
        (q, r) = queue.pop()
        if (q, r) in visited: continue
        if not condition_fn(q, r): continue
        visited.add((q, r))
        for neighbor in all_neighbors(q, r):
            if neighbor not in visited and is_in_map(neighbor):
                queue.append(neighbor)
    return visited
```

## 8. 设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 坐标系 | Axial | 算法简洁无特判，业界标准，文明6同方案 |
| 六边形朝向 | 尖顶（Pointy-Top） | 视觉稳定感强，上下对齐，符合战棋惯例 |
| 地图边界 | 矩形 | 实现简单，配合 active 标记可实现不规则边界 |
| 存储方式 | 字典 | 灵活支持不规则地图，查询 O(1) |
| Offset 偏移方案 | 奇数行右偏 | 尖顶六边形的标准方案，渲染简单 |
