# 09 城市概念

## 1. 什么是城市

城市是游戏中的核心经济单位。所有**资源产出**通过城市汇总，所有**单位招募**和**建筑建造**以城市为锚点。

```
每座城市产出 = 基础产出 + Σ 建筑加成 + Σ 改良产出
```

> 单元格本身不产出资源，单元格只提供"建改良的位置"。

## 2. 城市数据结构

```
City {
    id: CityID                       // 唯一标识
    name: string                     // 城市名称
    faction: FactionID               // 所属阵营

    // 城市属性
    level: int                       // 城市等级（影响基础产出）
    population: int                  // 当前人口
    max_population: int              // 人口上限

    // 产出（基础部分）
    base_output: dict<ResourceID, int>  // 基础产出 [待定]

    // 建筑
    buildings: list<BuildingID>      // 已建造的建筑列表
    wall_level: int                  // 城墙等级（0=无城墙，[待定] 最高等级）

    // 位置
    q: int                           // 市中心所在格 q 坐标
    r: int                           // 市中心所在格 r 坐标
    border_radius: int               // 边界半径（默认[待定]）
    border_cells: set<(int, int)>    // 边界内包含的所有格子坐标

    // 城防
    hp: int                          // 城市当前生命值
    max_hp: int                      // 城市最大生命值
    attack: int                      // 城市反击攻击力 [待定]
    defense: int                     // 城市防御力 [待定]

    // 城墙
    wall_hp: int                     // 城墙当前耐久（0=已破）
    max_wall_hp: int                 // 城墙最大耐久
    last_damaged_turn: int           // 城墙上次受损的回合号（用于修复判定）

    // 状态
    has_attacked: bool               // 本回合城墙是否已射击
    garrison_id: UnitID | None       // 驻军单位 ID（城市所在格上的单位）
}
```

## 4. 城市边界

### 4.1 边界定义

每个城市控制一片区域，区域内的单元格允许建造改良设施。

- 新建城市时默认边界半径：[待定]
- 边界内的单元格可建造改良
- 边界外的单元格不可建造改良

### 4.2 边界扩张

边界随人口增长扩张，扩张阈值和半径：[待定]。

### 4.3 边界冲突

- 两个城市的边界不能重叠
- 先建的城市优先占格
- 战争占领城市后边界重新计算

## 5. 城市战斗

### 5.1 城市自身的战斗能力

城市是一个独立的"战斗实体"，有自己的 HP、攻击力和防御力。

| 属性 | 说明 |
|------|------|
| max_hp | 城市最大生命值 [待定] |
| hp | 当前生命值，≤0 时城市陷落 |
| attack | 城市反击攻击力 [待定]（基于城市等级） |
| defense | 城市防御力 [待定]，减免所受伤害 |

**生命恢复**：城市每回合自动恢复 HP，恢复量 [待定]。

### 5.2 驻军

城市所在格上的单位即为该城市的驻军。由于单位不堆叠，一个城市同时只能有一个驻军。

- 驻军是**该格上的单位**，与城市是两个独立的战斗实体
- 驻军可正常移动出城（移动后该城市无驻军）
- 玩家可将单位移动进城市所在格，自动成为驻军

### 5.3 近战攻击城市

当敌方单位近战攻击城市时，按以下流程结算：

```
近战攻击城市(attacker, city):
    // 1. 攻击者对城市造成伤害
    damage_to_city = max(1, attacker.attack - city.defense)
    city.hp -= damage_to_city

    // 2. 城市反击（城市自身）
    damage_to_attacker = max(1, city.attack - attacker.defense)
    attacker.hp -= damage_to_attacker

    // 3. 驻军反击（如果有驻军）
    if city.garrison_id:
        garrison = get_unit(city.garrison_id)
        damage_to_attacker = max(1, garrison.attack - attacker.defense)
        attacker.hp -= damage_to_attacker

    // 4. 结算
    结算攻城结果(attacker, city)
```

即：近战攻城时，攻击者会受到**两次反击**——城市一次 + 驻军一次。

### 5.4 远程攻击城市

当敌方单位远程攻击城市时：

```
远程攻击城市(attacker, city):
    // 远程对城市伤害降低
    raw_damage = max(1, attacker.attack - city.defense)
    damage_to_city = floor(raw_damage × RANGED_DAMAGE_RATIO)  // [待定]，如 50%
    city.hp -= damage_to_city

    // 远程攻击不触发任何反击
```

远程攻城不触发城市反击和驻军反击。

### 5.5 城市陷落

当城市 HP ≤ 0 时：

1. 城市失去战斗能力，防御归零
2. 敌人可走进市中心格**占领城市**
3. 驻军（如果有）**直接死亡**（被移除）
4. 占领后城市归属转移，HP 恢复至 [待定]

### 5.6 城市被攻击于城墙

当城市拥有城墙（wall_level > 0）时，所有攻击**先作用于城墙**：

```
攻击有城墙的城市(attacker, city):
    if city.wall_hp > 0:
        // 伤害先打城墙
        wall_damage = calculate_wall_damage(attacker, city)
        city.wall_hp -= wall_damage
        if city.wall_hp <= 0:
            // 城墙被攻破，多余伤害不溢出到城市
            city.wall_hp = 0
            city.wall_level = 0  // 城墙被摧毁
    else:
        // 城墙已破，正常攻击城市
        normal_attack(attacker, city)
```

**城墙减伤规则**：

```
calculate_wall_damage(attacker, city):
    if attacker.type == SIEGE:
        // 攻城单位对城墙造成全额伤害
        return max(1, attacker.attack)
    else:
        // 非攻城单位对城墙伤害大幅降低
        return max(1, floor(attacker.attack × NON_SIEGE_RATIO))  // [待定]，如 20%
```

### 5.7 城墙修复

- 城墙连续 **3 回合**未受任何伤害后，可花费**石材**进行修复
- 修复量：[待定] 点耐久
- 修复不消耗行动力，在回合结算阶段自动执行（如有石材且满足条件）

### 5.8 城墙射击

拥有城墙的城市每回合可执行一次远程射击：

```
城墙射击(city, target):
    if city.has_attacked: return  // 本回合已射击
    if city.wall_hp <= 0: return  // 城墙已破，无法射击
    
    // 射击攻击力基于所属阵营的远程单位等级
    shot_attack = calculate_shot_attack(city.faction)  // [待定]
    damage = max(1, shot_attack - target.defense)
    target.hp -= damage
    city.has_attacked = true
```

- 射击范围：城市所在格为中心，射程 [待定]
- 攻击力基准：随国家远程科技/等级提升而提升（基础为弓箭手级别）
- 需要视线（受地形阻挡影响）
- 本回合已射击后不可再射击（has_attacked = true）

### 5.9 攻城战完整流程示例

```
回合 1：敌人近战单位 A 攻击有城墙的城市
        1. A 攻击 → 城墙受到 (A.atk × 20%) 伤害
        2. 城市反击 → A 受到 city.attack 伤害
        3. 驻军反击 → A 受到 garrison.attack 伤害
        4. A 仍存活，留在原位

回合 2：敌人攻城单位 B 攻击同一城市
        1. B 攻击 → 城墙受到 B.atk 全额伤害（城墙破）
        2. 城市反击 → B 受到 city.attack 伤害
        3. 驻军反击 → B 受到 garrison.attack 伤害
        4. B 仍存活

回合 3：敌人近战单位 A 再次攻击（城墙已破）
        1. A 攻击 → 城市受到 (A.atk - city.def) 伤害
        2. 城市反击 → A 受到 city.attack 伤害
        3. 驻军反击 → A 受到 garrison.attack 伤害
        4. A 死亡，城市 HP 降至 [低]

回合 4：敌人单位 C 攻击，城市 HP ≤ 0 → 城市陷落
        驻军死亡，C 进入城市格，城市归属转移
```

## 6. 产出汇总

```
城市总产出 = 基础产出 + Σ 建筑加成 + Σ 改良产出
```

- **基础产出**：由城市等级决定（见第 3 节）
- **建筑加成**：城市内建筑提供的各类加成（见 10-建筑概念）
- **改良产出**：边界内所有改良设施的产出之和（见 11-单元格改良）

## 7. 设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 单元格无独立产出 | 是 | 所有产出汇总到城市层面，简化计算 |
| 改良产出计入城市 | 是 | 改良是城市经济的一部分，非独立收入 |
| 近战攻城受两次反击 | 城市 + 驻军各一次 | 驻军有战术价值，攻城方需考虑代价 |
| 远程攻城伤害降低 | 是 | 远程白嫖城市不合理，需要近战或攻城单位配合 |
| 城墙独立 HP | 是 | 攻城必须先破墙，城墙有专门的减伤规则 |
| 非攻城单位对城墙伤害极低 | 是 | 避免步兵强拆城墙，强调攻城单位的必要性 |
| 城墙 3 回合未受伤可修复 | 是 | 修复窗口需要持续压制，否则攻城进度回退 |
| 城墙射击攻击力基于国家等级 | 是 | 鼓励科技升级，后期城市更有防御价值 |
