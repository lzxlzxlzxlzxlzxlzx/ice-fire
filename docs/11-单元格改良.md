# 11 单元格改良

## 1. 什么是改良

改良是建造在**地图单元格上**的设施，每个单元格最多一个改良。改良的产出计入所属城市。

与城市建筑的分工：

| 维度 | 改良（本文） | 建筑（10-建筑概念） |
|------|------------|-------------------|
| 位置 | 地图单元格上 | 城市内部 |
| 占地 | 占一整格 | 不占格 |
| 效果 | 产出资源 / 提供战术加成 | 影响城市面板属性 |
| 建造者 | 消耗城市资源 | 消耗城市资源 |
| 被摧毁 | 敌方可劫掠拆除 | 城市被占时可能被毁 |

## 2. 改良数据结构

```
Improvement {
    id: ImprovementID                // 唯一标识
    type: ImprovementType            // 改良类型（见第 3 节）
    
    // 产出定义（具体值 [待定]）
    output: dict<ResourceID, int>
    
    // 战术加成
    defense_bonus: int               // 守军防御加成
    sight_bonus: int                 // 视野加成
    heal_bonus: int                  // 单位每回合回血量
    
    // 建造成本（具体值 [待定]）
    cost: dict<ResourceID, int>
    
    // 地形限制
    allowed_terrains: list<TerrainType>
    allowed_features: list<FeatureType> | None  // 需要的地貌（如伐木场需森林）
}
```

## 3. 改良类型枚举

| 改良 | ID | 主要产出 | 战术加成 | 地形限制 |
|------|----|---------|---------|---------|
| 农场 | farm | 人力 | — | 平原 |
| 牧场 | pasture | 马匹 | — | 草原 |
| 伐木场 | lumber_mill | 木材 | — | 需森林地貌 |
| 矿井 | mine | 石材、钢铁 | — | 丘陵、山脉 |
| 渔港 | fishing_wharf | 金币、人力 | — | 沿海、湖泊 |
| 种植园 | plantation | 金币 | — | 丘陵、平原 |
| 采石场 | quarry | 石材 | — | 丘陵、山脉 |
| 哨塔 | watchtower | — | 视野 +[待定] | 任意 |
| 要塞 | fort | — | 防御 +[待定]，回血 +[待定] | 任意 |
| 道路 | road | — | 相连格移动消耗 -[待定] | 任意 |
| 营地 | camp | — | 回血 +[待定] | 任意 |

> 各类改良的具体产出值、建造成本 [待定]，待经济系统平衡时填写。

## 4. 建造与摧毁规则

**建造**：
- 只能在所属城市边界内的单元格上建造
- 每个单元格最多 1 个改良
- 建造消耗城市资源

**地形限制**：
- 改良能否建在某个格上，由该格的基础地形和地貌共同决定
- 例如：伐木场只能建在有森林地貌的格上；牧场只能建在草原上

**摧毁**：
- 敌方部队移动到改良格可执行劫掠（消耗 1 移动力）
- 改良被摧毁后需重建
- 城市被占领时，其边界内所有改良归属敌方

---

## 5. 设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 改良产出计入城市 | 是 | 所有产出汇总到城市层面，不追踪单格收入 |
| 地形限制 | 白名单制 | 确保只有合理的改良-地形组合 |
| 改良可被劫掠 | 是 | 劫掠是战争经济的重要组成，铁群岛核心玩法 |
